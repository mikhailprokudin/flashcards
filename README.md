# 汉字卡

**Карточки для изучения китайского языка** — веб-приложение на Vue 3 с сервером на Node.js и MySQL. Слова и прогресс хранятся в аккаунте; повторения ведутся по упрощённой SRS-модели.

---

## Возможности

### Аккаунт и синхронизация

- Регистрация и вход по email и паролю, JWT для запросов к API.
- Данные колод и карточек привязаны к пользователю и подгружаются с сервера.

### Колоды

- Создание, просмотр, переименование и удаление колод.
- В каждой колоде — список карточек, быстрый переход к повторению только этой колоды.

### Карточки

Поля карточки: **汉字**, **拼音**, **перевод**, пример использования, заметки.

- **Новая карточка**: можно вводить текст вручную или **нарисовать иероглиф** на поле ввода — распознавание рукописи и выбор кандидата, затем сбор слова по частям; после выбора иероглифов доступна **подсказка пиньиня и значения** по введённому слову.
- **Редактирование**: для поля 汉字 доступен **пошаговый разбор почерка** (Hanzi Writer) по эталонным чертам.

### Повторение (SRS)

- Очередь карточек к повторению с лимитами (новые и общий размер очереди запрашиваются с API).
- Повторение **по всем колодам сразу** или **в рамках одной колоды**.
- Ответы «знаю» / «не знаю» обновляют интервалы на сервере.

**Режим «тройной квиз»** 🎯 (по умолчанию включён; можно выключить в настройках):

- Для каждой карточки в сессии три типа шагов: **перевод**, **только иероглифы**, **написание по чертам** без подсказки с лицевой стороны.
- Шаги **перемешиваются между разными словами**; один итоговый ответ SRS фиксируется после завершения всех трёх шагов по карточке.
- Управление: тап по карточке — переворот, свайп влево/вправо после переворота — ответ.

**Классический режим** (если тройной квиз выключен):

- Переворот карточки тапом; после переворота — свайпы для ответа.
- Опционально в настройках: **обязательный почерк (笔顺)** после переворота — нужно написать каждый иероглиф по чертам (данные черт могут подгружаться с CDN, если локально их нет).

### Главная и статистика

- Сводка: сколько карточек **без повторений**, **в процессе изучения**, **усвоено**, есть ли сейчас что повторять.
- **Серия дней** с занятиями (текущая и лучшая).

### Интерфейс и PWA

- Таб-бар для авторизованных пользователей, экран загрузки, переходы между страницами.
- **Web App Manifest**, иконки и service worker (Workbox): установка на домашний экран и автономное обновление кэша сборки. Без сети открывается интерфейс и отдаются уже закэшированные статические ресурсы (в том числе шрифты Google и файлы с jsDelivr для почерка); вход, колоды и SRS по-прежнему требуют доступ к API.
- Шрифт Noto Sans SC для корректного отображения китайских символов.

### Прочее

- Озвучивание китайского текста там, где это уместно в интерфейсе повторения.

## Локальная разработка

```bash
npm install
npm run dev
```

Фронтенд в режиме разработки обычно открывается на `http://localhost:5173`. Запросы к `/api` проксируются на API (по умолчанию `http://127.0.0.1:3000`; при необходимости задайте `VITE_API_PROXY_TARGET` в `.env`, см. [`.env.example`](.env.example)).

Сборка и проверка типов:

```bash
npm run build
```

Тесты (Node test runner + tsx):

```bash
npm test
```

Для production-сборки при необходимости укажите полный origin API в `VITE_API_BASE_URL` (без завершающего слэша) — см. комментарии в `.env.example`.

## Бэкенд API (Docker) 🐳

Нужны [Docker](https://docs.docker.com/get-docker/) и Docker Compose v2.

1. Скопируйте [`.env.example`](.env.example) в `.env` и задайте свои значения: пароли MySQL (`MYSQL_PASSWORD`, `MYSQL_ROOT_PASSWORD`), тот же пароль пользователя в `DATABASE_URL`, случайную строку для `JWT_SECRET`.
2. Из корня репозитория:

   ```bash
   docker compose up --build
   ```

   Поднимутся MySQL (с healthcheck) и API: при старте контейнера API выполняются миграции, затем сервер слушает порт **3000** внутри контейнера; на хост пробрасывается порт из переменной `PORT` в `.env` (по умолчанию 3000).

3. Проверка: `GET http://localhost:3000/health` → `{"ok":true}` (подставьте свой порт, если меняли `PORT`).

В `.env` для API задайте `CORS_ORIGIN` под origin вашего фронтенда (например `http://localhost:5173`).

### Пример регистрации и входа

Тело запроса — JSON, в ответе приходит `token`:

```bash
curl -sS -X POST http://localhost:3000/auth/register \
  -H 'Content-Type: application/json' \
  -d '{"email":"user@example.com","password":"password123"}'

curl -sS -X POST http://localhost:3000/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"user@example.com","password":"password123"}'
```

Дальнейшие запросы к защищённым маршрутам: заголовок `Authorization: Bearer <token>`.
